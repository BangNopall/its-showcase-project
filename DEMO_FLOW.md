# DEMO_FLOW.md

## Project Session Demo Flow

**Project:** Website BEM FILKOM UB Showcase  
**Presented by:** IT Solutions Bureau — SGE/BEM FILKOM UB  
**Audience:** President University  
**Session Type:** Project Session / Product Showcase  
**Main Goal:** Menampilkan Website BEM FILKOM UB sebagai platform digital terintegrasi untuk pusat informasi, publikasi berita, pengajuan event, approval workflow, dan pendaftaran kegiatan.

---

## 1. Demo Objective

Demo ini bertujuan untuk menunjukkan bahwa Website BEM FILKOM UB bukan hanya website profil organisasi, tetapi juga sistem digital yang membantu proses kerja internal BEM FILKOM UB.

Melalui demo ini, audience diharapkan memahami:

1. Website BEM FILKOM UB berfungsi sebagai pusat informasi resmi organisasi.
2. Website mendukung publikasi berita dan informasi kegiatan.
3. Website menyediakan sistem pengajuan event secara digital.
4. Website memiliki alur preview, submit, approval, dan tracking status event.
5. Website dapat mendukung kebutuhan open recruitment dan pendaftaran peserta.
6. Website menjadi bentuk digitalisasi proses kerja organisasi oleh IT Solutions Bureau.

---

## 2. Demo Style

Demo dilakukan dengan gaya **product showcase**, bukan presentasi laporan biasa.

Tone yang digunakan:

- Profesional
- Modern
- Singkat dan jelas
- Fokus pada fitur dan value
- Menunjukkan alur penggunaan dari sisi user dan admin

Presentasi tidak perlu terlalu teknis dari awal. Mulai dari masalah, solusi, fitur, lalu masuk ke live demo.

# 11. Quick Demo Script — 10 Minutes Version

Gunakan jika waktu presentasi lebih panjang.

```txt
Selamat siang teman-teman dari President University. Kami dari IT Solutions Bureau BEM FILKOM UB akan memperkenalkan salah satu project digital yang kami kembangkan, yaitu Website BEM FILKOM UB.

Website ini kami rancang bukan hanya sebagai media informasi organisasi, tetapi juga sebagai platform digital terintegrasi untuk mendukung pengelolaan event, publikasi berita, dan komunikasi antara organisasi dengan mahasiswa.

Sebelumnya, terdapat beberapa tantangan dalam pengelolaan informasi dan kegiatan organisasi. Informasi dapat tersebar di berbagai kanal, proses pengajuan event masih manual, approval belum sepenuhnya mudah dipantau, dan publikasi kegiatan membutuhkan sistem yang lebih rapi.

Dari kebutuhan tersebut, kami mengembangkan Website BEM FILKOM UB sebagai platform yang memiliki dua fokus utama: menjadi pusat informasi resmi organisasi dan membantu digitalisasi proses event management.

Pertama, kita mulai dari homepage. Homepage ini menjadi pintu utama bagi mahasiswa dan publik untuk mengenal BEM FILKOM UB, melihat informasi terbaru, serta mengakses halaman event dan berita. Di halaman ini terdapat navigasi utama, latest events, latest news, dan footer yang mengarah ke arsip website BEM dari tahun sebelumnya.

Selanjutnya, kita masuk ke halaman News. Pada halaman ini, user dapat membaca publikasi resmi dari BEM FILKOM UB. Fitur yang tersedia meliputi search news, filter kategori, highlight news, list latest news, dan pagination. Dengan fitur ini, informasi organisasi dapat lebih mudah dicari, dibaca, dan diarsipkan.

Setiap berita juga memiliki halaman detail. Pada halaman detail, user dapat melihat judul, author, tanggal publikasi, kategori, gambar utama, isi artikel, tombol share, dan rekomendasi berita lainnya. Ini membuat publikasi organisasi terlihat lebih profesional dan mudah dibagikan.

Selain publikasi informasi, website ini juga mendukung kebutuhan event dan open recruitment. Pada halaman preview event, admin dapat melihat bagaimana event akan ditampilkan sebelum dikirim. Terdapat informasi seperti judul event, deskripsi, division needs, countdown registration, timeline, dan registration flow.

Fitur preview ini penting karena membantu admin memastikan data yang dimasukkan sudah benar dan rapi sebelum diajukan. Hal ini dapat mengurangi kesalahan input dan revisi berulang.

Setelah event berjalan, admin dapat mengakses dashboard detail event. Pada dashboard ini, admin dapat melihat summary participant, status peserta, melakukan pencarian berdasarkan nama atau NIM, melakukan filter division, mengekspor data ke Excel, dan membuka detail participant.

Dashboard ini membantu proses pengelolaan peserta menjadi lebih terstruktur dan efisien dibandingkan pencatatan manual.

Secara role, website ini membagi akses menjadi beberapa bagian. General user dapat mengakses berita, event, dan form pendaftaran. Admin dari Ministry atau Bureau dapat mengajukan event dan mengelola participant. Super Admin dari IT Solutions dapat melakukan review dan approval terhadap pengajuan event.

Dari sisi pengembangan, project ini dibangun secara bertahap melalui milestone. Milestone pertama berfokus pada fondasi sistem seperti UI/UX, landing page, dashboard, login, authentication, dan role-based access. Milestone kedua berfokus pada fitur inti seperti event request, approval flow, open recruitment, news management, API integration, dan final testing.

Dampak dari project ini adalah informasi organisasi menjadi lebih terpusat, pengajuan event lebih terstruktur, approval lebih transparan, publikasi lebih profesional, dan pengelolaan participant menjadi lebih mudah.
```
