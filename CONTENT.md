## Opening
Website BEM FILKOM UB dikembangkan sebagai platform digital terintegrasi yang berfungsi sebagai pusat informasi, pengelolaan kegiatan, serta media komunikasi antara organisasi dan mahasiswa.

## Background
Website BEM Fakultas dibutuhkan sebagai pusat informasi dan sistem terintegrasi untuk mendukung kegiatan organisasi, khususnya dalam pengelolaan event dan komunikasi kepada mahasiswa. Sebelumnya, proses pengajuan event masih manual dan tidak terstruktur, menyebabkan approval lambat, status pengajuan kurang transparan, dan publikasi event kurang efisien.

## Problem
- Informasi organisasi belum terpusat.
- Pengajuan event masih manual.
- Approval event membutuhkan alur yang lebih jelas.
- Status pengajuan belum transparan.
- Publikasi berita dan event perlu sistem yang lebih mudah dikelola.
- Mahasiswa membutuhkan akses informasi yang cepat dan responsif.

## Objective
- Mempermudah proses pengajuan dan approval event.
- Menyediakan platform terpusat untuk publikasi kegiatan.
- Meningkatkan efisiensi koordinasi internal organisasi.
- Menyediakan informasi yang jelas dan mudah diakses mahasiswa.
- Meningkatkan transparansi status pengajuan event.
- Memperkuat digital branding BEM FILKOM UB.

## Product Vision
Menjadikan Website BEM FILKOM UB sebagai pusat informasi digital dan sistem pengelolaan kegiatan yang terintegrasi, transparan, dan mudah digunakan oleh seluruh elemen organisasi serta mahasiswa FILKOM UB.

## Target Users

### Admin / Ministry & Bureau
Goal: Mengajukan event dengan cepat dan jelas.  
Pain Point: Proses manual ribet dan revisi bolak-balik.  
Needs: Form submission, preview, tracking status, dan feedback.

### Super Admin / IT Solutions
Goal: Review event dengan efisien.  
Pain Point: Banyak request dan sulit tracking status.  
Needs: Dashboard management, approval workflow, dan monitoring.

### General User / Mahasiswa
Goal: Cari informasi event, berita, dan daftar open recruitment.  
Pain Point: Informasi tersebar dan sulit diakses.  
Needs: Website yang cepat, simpel, dan mobile-friendly.

## Main Solution
Website BEM FILKOM UB adalah platform digital terintegrasi yang menjadi pusat informasi resmi organisasi, menyediakan sistem pengajuan event, dashboard approval, publikasi berita, serta fitur pendaftaran event atau open recruitment.

## Features

### Landing Page
Halaman utama yang menampilkan identitas BEM FILKOM UB, highlight informasi, navigasi utama, dan akses ke berita atau event.

### Event Request Management
Fitur untuk admin Ministry/Bureau agar dapat mengajukan event secara digital melalui structured submission form.

### Preview Before Submit
Admin dapat melihat preview tampilan event sebelum submit untuk mengurangi kesalahan input dan revisi.

### Approval Workflow
Super Admin dapat melakukan review event dengan status Approved, Rejected, atau Need Revision disertai feedback.

### Admin & Super Admin Dashboard
Dashboard digunakan untuk mengelola event request, melihat status pengajuan, memberikan feedback, dan melakukan monitoring.

### Open Recruitment / Registration
Mahasiswa dapat mengakses halaman detail event atau open recruitment dan melakukan pendaftaran melalui form yang tersedia.

### News Publication
Fitur berita untuk publikasi informasi dan dokumentasi kegiatan BEM FILKOM UB, mencakup list berita, detail berita, search-filter, tag, share, pinned/highlight, schedule, draft, publish, pagination, dan summary dashboard.

### Role-Based Access Control
Sistem membedakan hak akses antara Super Admin, Admin, dan General User agar data tetap aman dan fitur sesuai kebutuhan role.

### Responsive Website
Website dirancang responsif agar dapat digunakan dengan nyaman melalui desktop maupun mobile.

## Tech Stack
- Frontend: Next.js, React, Tailwind CSS
- Animation: Framer Motion, Spline 3D
- Backend: REST API, Authentication API, Event Management API, News Management API
- Database: Relational Database / Centralized Data Repository
- Project Management: Notion Timeline, GitHub, Sprint-based Development
- Deployment: https://bemfilkom.ub.ac.id

## Development Process

### Milestone 1: Dasar Sistem & Autentikasi
Fokus pada UI/UX, slicing landing page, dashboard user/admin/super admin, login page, authentication, dan role-based access control.

### Milestone 2: Fitur Inti & Finalisasi
Fokus pada event request management, approval flow, open recruitment, news management, API integration, dan final testing.

## Progress Highlight
Completed features include search and filter berita, pagination tabel berita, summary cards, tag interaction, empty state, event registration UI/UX, dashboard admin/user/super admin, login/logout API, event request API, approval API, registrant API, news API, publish/draft/schedule berita, pinned berita, share berita, create/update/delete berita, cron job auto publish, and middleware.
